# パワースポット記事自動投稿システム

Claude Projects で高品質な記事を作成し、Claude Code で WordPress に自動投稿するシステムです。

## 🎯 ワークフロー

```
【Claude Projects】
高品質な記事を作成（対話しながら調整可能）
  ↓ Markdownで保存
【GitHub/ローカル】
articles/ ディレクトリに保存
  ↓
【Claude Code】
WordPressに自動投稿（数秒）
```

---

## 💰 コスト

- **記事作成**: $0（Claude Projects Proプラン内）
- **投稿**: $0（Claude Codeのトークンのみ）
- **合計**: **完全無料**

---

## 📝 使い方

### Step 1: Claude Projects で記事作成

1. **Claude Projects を開く**
   - https://claude.ai/projects

2. **新しいプロジェクトを作成**
   - プロジェクト名: 「パワースポット記事作成」

3. **以下のプロンプトを使用:**

\`\`\`
あなたは全国300ヶ所以上のパワースポットを訪れたベテランライターです。

パワースポット「[スポット名]」について、以下の形式でMarkdown記事を作成してください:

---
title: 【[スポット名]】パワースポット完全ガイド｜ご利益とアクセス（50文字以内）
excerpt: [スポット名]の魅力を要約（120文字以内）
---

# [スポット名]のパワースポット完全ガイド

## 基本情報
- 正式名称
- 所在地
- 創建年・歴史
- 祭神・本尊

## パワースポットとしての特徴
- エネルギーの種類
- 実際に訪れた時の体験（具体的に）
- 他のスポットとの違い

## ご利益
- 主なご利益
- 具体的な効果
- 口コミや体験談

## 見どころ
- 境内の主要スポット
- 隠れた名所
- フォトスポット
- 訪問時の具体的な様子

## アクセス方法
- 最寄り駅・バス停
- 車でのアクセス
- 駐車場情報
- 所要時間

## 訪問のベストタイミング
- おすすめの時期・時間帯
- 混雑を避けるコツ
- 季節ごとの見どころ

## 周辺情報
- 近くのおすすめスポット
- 地元グルメ
- 宿泊施設

## まとめ
記事の総括

---

**重要な要件:**
1. 実際に訪れたかのような具体的な描写を含める
2. 一般的な情報だけでなく、独自の視点を入れる
3. 「〜と言われています」より「〜です」と断定的に
4. 時間帯や季節による雰囲気の違いを描写
5. 失敗談や注意点も含める
6. 2000文字以上
7. SEOキーワードを自然に含める
\`\`\`

4. **記事が生成されたら調整**
   - 「もっと具体的に」
   - 「この部分を詳しく」
   - 「独自性を出して」
   など対話しながら改善

5. **完成したら `articles/` に保存**
   - ファイル名例: `izumo-taisha.md`, `itsukushima.md`

---

### Step 2: WordPress に投稿

1. **Claude Code を開く**

2. **以下のコマンドを実行:**

\`\`\`bash
cd /c/Users/user/work/powerspot-content-generator
node post-from-markdown.js articles/izumo-taisha.md
\`\`\`

3. **WordPress管理画面で確認**
   - 下書きとして投稿されます
   - 内容を確認して公開

## 🎨 記事のカスタマイズ

### タイトルの工夫
- ❌ 悪い例: 「伊勢神宮について」
- ✅ 良い例: 「【伊勢神宮】日本最高峰のパワースポット完全ガイド｜ご利益とアクセス」

### 本文の工夫
- 具体的な時間帯を書く（「早朝5時」「夕方17時」）
- 季節を明記（「2024年3月の桜の季節」）
- 五感を使った描写（「清々しい空気」「鳥のさえずり」）
- 実際の体験談風に（「訪れた際、〜と感じました」）

## 🔧 トラブルシューティング

### 投稿に失敗する
\`\`\`bash
# 接続テスト
node test-connection.js
\`\`\`

### Markdownの形式が正しいか確認
Front matter（メタデータ）が必須:
\`\`\`markdown
---
title: 記事タイトル
excerpt: 要約
---

本文...
\`\`\`

### デザインが反映されない
WordPress側のテーマ設定を確認してください。
