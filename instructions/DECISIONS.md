## 記事構造に関する判断

### セクション順序の変更（2024年11月17日）

#### 決定内容

**基本情報セクション（アクセス・営業時間）を、「参拝・見学ガイド」の後に配置する**

#### 変更前の順序
```
1. 導入部
2. スポットの基本情報 ← アクセス・営業時間がここにあった
3. このスポットの魅力
4. ご利益
5. ベストな訪問時期
6. 参拝・見学ガイド
7. 周辺情報
...
```

#### 変更後の順序
```
1. 導入部
2. このスポットの魅力
3. ご利益・期待できる効果
4. ベストな訪問時期
5. 参拝・見学ガイド
6. スポットの基本情報 ← ここに移動
7. 周辺情報
8. 訪問者の口コミ
9. よくある質問
10. まとめ
```

#### 判断理由

**ユーザーの心理的フロー**:

1. **導入部**: 「このスポット、気になる」（興味喚起）
2. **魅力**: 「どんな場所なんだろう？」（関心増大）
3. **ご利益**: 「自分に合ってるかも」（自分ごと化）
4. **訪問時期**: 「いつ行こうかな」（具体的検討）
5. **参拝ガイド**: 「どう参拝すればいいの？」（実行イメージ）
6. **基本情報**: 「じゃあアクセスは？営業時間は？」← ここで実用情報
7. **周辺情報**: 「ランチはどこで食べよう」（旅行計画）

**変更前の問題点**:
- 興味が薄い段階で実用情報（アクセス・営業時間）を提示
- 「まだ行くかわからないのに、アクセス見ても意味ない」と離脱
- 魅力を伝える前に読者が去る

**変更後のメリット**:
- 興味が高まってから実用情報を提示
- 「行きたい！じゃあどうやって行くの？」の流れで自然
- 離脱率低下、読了率向上

#### 影響範囲

- `instructions/ARTICLE_GENERATION_MASTER.md` セクション番号変更
- `articles/izumo-taisha.md` セクション移動済み
- 今後生成するすべての記事

#### 検証方法

今後、記事生成時に以下を確認：
- 基本情報が「参拝・見学ガイド」の後にあるか
- セクション番号が正しいか（7番目）

## CTAに関する判断

### CTAの最小化（2024年11月17日）

#### 決定内容

**記事末尾の診断CTAは、控えめなテキストリンクのみとする**

#### 変更前（大きなCTAセクション）

```markdown
## 「出雲大社、行ってみたい！」と思ったあなたへ

実は、パワースポットには**「人それぞれの相性」**があることをご存知ですか？

同じスポットでも、訪れる人の生まれ持った特性によって、得られる効果や感じ方は大きく異なります。

あなたの生年月日や名前から導き出される「縁のタイプ」を知ることで、**あなたに最も合ったパワースポット**を見つけることができます。

### 【3分で無料診断】あなただけの「縁タイプ」を知る

診断では以下がわかります：
- あなたの縁プロファイル（60分類×12位置）
- 生まれ持ったエネルギー特性
- 名前に込められた縁の響き
- 現在のバイオリズム状態

**診断は完全無料です**

さらに、診断後には、あなたに最適な**全国のパワースポット18箇所**を完全網羅した詳細レポートをお申し込みいただけます。

**詳細レポートに含まれる内容**:
- 運命の開運スポット（2箇所）
- 金運アップスポット（2箇所）
- 恋愛成就スポット（2箇所）
[...]

**15,000字超の超詳細レポート**
**料金**: ¥2,980（税込）

→ **無料診断を始める**
```

#### 変更後（控えめなリンク）

```markdown
---

この記事があなたの出雲大社参拝のお役に立てれば幸いです。良いご縁に恵まれますように。

※本記事の情報は2025年11月時点のものです。訪問前に必ず公式サイトで最新情報をご確認ください。
```

HTMLフッターに自動追加される控えめなリンク：
```html
💡 あなたに最適なパワースポットを知りたい方は
無料の相性診断をお試しください（3分で完了）
```

#### 判断理由

**問題点（変更前）**:

1. **文脈の断絶**:
   - パワースポット記事（スポット情報を求める読者）
   - → 突然、診断アプリへ誘導
   - 読者: 「は？なんでいきなり診断？」

2. **有料商品の早期提示**:
   - 記事末尾でいきなり「¥2,980」
   - 読者: 「結局これ売りたいだけ？」
   - 信頼感低下

3. **記事の価値低下**:
   - 「価値提供」ではなく「セールス」に見える
   - SEO記事としての純粋性が失われる

4. **サイト全体との関係性**:
   - 診断訴求はヘッダー・サイドバーで常に表示されるべき
   - 記事末尾で重複して訴求する必要なし

**解決策**:

**記事の役割を明確化**:
- 記事 = **価値提供＋SEO集客**（診断への直接誘導はしない）
- サイト全体 = **診断訴求**（ヘッダー・サイドバー・固定バナー）

**診断への自然な流れ**:
```
SEO検索
  ↓
記事閲読（価値提供・信頼構築）
  ↓
サイト内回遊（他の記事も見る）
  ↓
ヘッダー/サイドバーの診断バナーが気になる
  ↓
診断実施
  ↓
有料レポート購入
```

**記事末尾の控えめなリンク**:
- 押し付けがましくない
- 興味ある人だけクリック
- 記事の価値を損なわない

#### 影響範囲

- `post-from-markdown-styled.js` の `addFooterCTA()` 関数を簡素化
- `generate-html-preview.js` の `addFooterCTA()` 関数を簡素化
- `articles/izumo-taisha.md` のまとめセクションを簡素化

#### 検証方法

- 記事末尾に大きなCTAセクションがないことを確認
- HTMLフッターに控えめなリンクがあることを確認
- 有料商品（¥2,980）への言及がないことを確認

## 文字数に関する判断

### 文字数範囲の変更（2024年11月17日）

#### 決定内容

**記事の文字数を 4,500-5,000文字（厳守）とする**

#### 変更前
- 3,500-5,000文字（幅1,500字）

#### 変更後
- **4,500-5,000文字（幅500字）**

#### 判断理由

**SEO効果**:
- Google検索で上位表示される記事は平均4,000-5,000字
- 競合サイトとの差別化（薄いコンテンツとの差別化）
- E-A-T（専門性・権威性・信頼性）の向上

**ユーザー満足度**:
- 充実した情報量（周辺グルメ、モデルコース等）
- 一度の訪問で必要な情報がすべて得られる
- リピート訪問・ブックマークされやすい

**ビジネス効果**:
- 滞在時間増加（Googleアルゴリズムで評価）
- 信頼感向上（診断アプリへの誘導効果UP）
- SNSシェアされやすい（詳細で役立つ記事）

**3,500字では不足する理由**:
- 周辺グルメ3-5店舗を詳しく書くと不足
- モデルコースを具体的に書くと不足
- 体験談・エピソードを充実させると不足

#### 影響範囲

- `instructions/ARTICLE_GENERATION_MASTER.md` 文字数指定を4,500-5,000に変更
- `powerspot_article_prompt_v2.md` 文字数指定を3箇所変更
- 今後生成するすべての記事

#### 検証方法

生成後、必ず文字数カウント：
```bash
# Markdown文字数カウント（見出し・記号含む）
wc -m articles/izumo-taisha.md
```

4,500-5,000字の範囲内であることを確認。

## 禁止事項に関する判断

### 五行理論・相性判断の完全排除（2024年11月16日）

#### 決定内容

**記事では五行理論・相性判断に一切触れない**

#### 禁止事項リスト

❌ **絶対に書いてはいけないこと**:
1. 五行理論の説明（木火土金水）
2. 12位置システムの言及
3. 60分類・縁タイプの詳細
4. 「○○タイプの人におすすめ」
5. 生年月日や名前との相性分析
6. バイオリズムとの関連
7. 統計学的根拠・計算式

#### 判断理由

**1. 読者の目的とのズレ**
- 読者が知りたいこと: 「出雲大社ってどんな場所？」「アクセスは？」
- 五行理論: 専門的すぎて興味がない
- 結果: 離脱

**2. 怪しいサイトと思われるリスク**
- 「○○タイプの人におすすめ」→ 占いサイト？
- 生年月日で相性判断 → 怪しい...
- 結果: 信頼感低下、SEO評価低下

**3. 診断アプリの価値低下**
- 記事で相性の仕組みを説明してしまう
- 「記事で無料で読めるなら、診断不要では？」
- 結果: コンバージョン低下

**4. SEO効果の低下**
- ユーザーの検索意図: 「出雲大社 パワースポット」= スポット情報
- 五行理論の説明: ユーザーの検索意図とズレる
- 結果: Googleの評価低下

#### 正しいアプローチ

**記事で書くべきこと**:
- ✅ 「出雲大社は縁結びのご利益があるとされています」（一般論）
- ✅ 「包み込むような温かいエネルギーを感じます」（感覚的）
- ✅ 「訪れた方の多くが良縁に恵まれたと語っています」（体験談）

**診断アプリで提供すること**:
- 五行理論に基づく相性分析
- 60分類×12位置の詳細
- 「あなたに合うスポット18箇所」

**役割分担**:
- 記事 = 一般的なスポット情報（万人向け）
- 診断 = パーソナライズされた情報（個別最適化）

#### 検証方法

記事生成後、以下のキーワードが含まれていないことを確認：
- 「五行」「木火土金水」
- 「12位置」「60分類」
- 「縁タイプ」
- 「○○タイプの人におすすめ」
- 「生年月日」「名前との相性」
- 「バイオリズム」
- 「統計学」

## 品質基準に関する判断

### 具体性を重視する方針（2024年11月16日）

#### 決定内容

**曖昧な表現を排除し、具体的な数字・時刻・描写を豊富に含める**

#### 品質基準

**必須要件**:
- 具体的な数字: **15箇所以上**
- 具体的な時刻: **5箇所以上**
- 五感描写: **3箇所以上**
- 体験談・エピソード: **2-3箇所**

#### 悪い例 vs 良い例

**❌ 悪い例（曖昧）**:
```
早朝がおすすめです。美しい景色が広がっています。
近い場所にあります。パワーが強いです。
```

**✅ 良い例（具体的）**:
```
午前5時〜7時が最も空いており、神聖な雰囲気を独り占めできます。
早朝5時、朝霧に包まれた境内。鳥居の向こうから差し込む朝日が、
樹齢300年の杉並木を黄金色に染め上げます。足元の玉砂利を踏む音だけが
静寂を破り、どこからか梅の香りが漂ってきます。

駅から徒歩12分、緩やかな上り坂を進みます。

神楽殿の大注連縄は長さ13.6メートル、重さ5.2トン。
```

#### 判断理由

**SEO効果**:
- 具体的な情報 = オリジナルコンテンツ
- 「5時」「12分」「13.6メートル」= 他サイトにない独自性
- Google: 「このサイトは詳しい！」→ 上位表示

**ユーザー満足度**:
- 「早朝」だけでは役に立たない
- 「5時〜7時」なら計画できる
- 「駅から近い」より「徒歩12分」の方が親切

**信頼感向上**:
- 具体的 = 実際に訪れた証拠
- 曖昧 = コピペ記事っぽい

**競合との差別化**:
- 他サイト: 「美しい景色」（誰でも書ける）
- 当サイト: 「樹齢300年の杉並木を黄金色に染め上げます」（オリジナル）

#### 検証方法

記事生成後、以下をカウント：
1. 数字の出現回数（15回以上か？）
2. 時刻の出現回数（5回以上か？）
3. 五感描写の箇所数（3箇所以上か？）
4. 体験談の箇所数（2-3箇所あるか？）

`instructions/quality-checklist.md` でチェック。

## 技術実装に関する判断

### Markdown + スタイル付きHTML方式の採用（2024年11月16日）

#### 決定内容

**記事はMarkdownで保存し、投稿時にスタイル付きHTMLに変換する**

#### アーキテクチャ

```
1. 記事作成（Markdown）
   ↓
2. HTMLプレビュー生成（ローカル確認）
   ↓
3. WordPress投稿（スタイル付きHTML）
```

#### 技術的選択

**Markdownで保存する理由**:
- 可読性が高い（人間が読みやすい）
- 編集が簡単（テキストエディタで編集可能）
- バージョン管理に適している（Git差分が見やすい）
- プラットフォーム非依存（WordPressに依存しない）

**HTMLに変換する理由**:
- WordPressのビジュアルエディタで正しく表示
- セクションごとのスタイリングが可能
- 色分け・ボックス・絵文字で読みやすく

**インラインCSSを使う理由**:
- WordPressテーマに依存しない
- テーマ変更時も崩れない
- メール配信時も正しく表示（将来的な拡張）

#### ファイル構成

```javascript
// post-from-markdown-styled.js
function markdownToHtml(markdown) {
  // セクションを分割
  let sections = markdown.split(/(?=^## )/gm);

  // セクションごとにスタイル適用
  sections.forEach((section) => {
    if (sectionTitle.includes('基本情報')) {
      sectionStyle = styles.infoBox; // 青系ボックス
    } else if (sectionTitle.includes('口コミ')) {
      sectionStyle = styles.reviewBox; // 引用スタイル
    }
    // ...
  });
}
```

#### 判断理由

**メンテナンス性**:
- Markdown = 人間が編集しやすい
- HTMLに変換 = 機械的に処理

**移植性**:
- 将来的に別のCMSに移行可能
- Markdownファイルがあればどこでも使える

**プレビュー機能**:
- ローカルでHTMLプレビュー可能
- WordPress投稿前に確認できる

#### 代替案と却下理由

**❌ 案1: HTMLで直接保存**
- 編集が大変（タグだらけ）
- 差分が見づらい（Git）

**❌ 案2: WordPressで直接執筆**
- バージョン管理できない
- ローカルにバックアップがない
- Claude Codeで生成しにくい

**✅ 採用案: Markdown + 変換スクリプト**
- 編集しやすい
- バージョン管理可能
- プレビュー可能
- 移植性高い
