# 記事作成インストラクション

このディレクトリには、高品質なパワースポット記事を作成するためのガイドラインが含まれています。

## 📁 ファイル構成

### 1. `writing-guidelines.md`（最重要）
記事を書く際に**必ず読み込むべき**ガイドライン。以下が含まれています：

- 文体とトーン
- 各セクションの書き方（詳細）
- 具体性を高める技法
- 独自性を出す方法
- SEO対策
- 避けるべき表現

**使い方**: 記事を書く前に必ず全文を読み、基準を理解してから執筆する

### 2. `quality-checklist.md`
記事完成後の品質チェック用リスト。以下が含まれています：

- 基本要件のチェック項目
- 文章品質の確認事項
- 独自性・差別化のチェック
- SEO最適化の確認
- 100点満点の自己評価シート

**使い方**: 記事を書き終えたら、このチェックリストで評価し、90点以上になるまで修正

### 3. `example-high-quality.md`
実際の高品質記事の見本（出雲大社の例）。以下の要素を含む模範例：

- 具体的な数字・時刻（50箇所以上）
- 五感を使った描写
- 独自の視点・体験談
- 実用的なアドバイス
- 自然なSEO対策
- 文字数: 約4,200文字

**使い方**: 書き方に迷ったら参照。この品質を目指す

## 🚀 記事作成フロー

### ステップ1: 準備
```
1. `writing-guidelines.md` を全文読む
2. `example-high-quality.md` を確認して品質基準を理解
3. 対象のパワースポットについて基本情報を調査
```

### ステップ2: 執筆
```
1. Front matterから書き始める（title, excerpt）
2. 必須セクションを順番に埋めていく
3. ガイドラインに従って、具体的・独自的・実用的に書く
4. 目標文字数: 2,500-3,500文字
```

### ステップ3: 品質チェック
```
1. `quality-checklist.md` で自己評価
2. 90点未満の場合、低い項目を重点的に修正
3. 90点以上になるまで繰り返す
```

### ステップ4: 公開
```
1. `articles/スポット名.md` として保存
2. `node post-from-markdown.js articles/スポット名.md` で投稿
```

## 📊 品質基準（最低ライン）

記事が以下の基準を満たしていることを確認：

| 項目 | 基準 | 確認 |
|-----|------|------|
| 文字数 | 2,500文字以上 | □ |
| 具体的な数字 | 5箇所以上 | □ |
| 具体的な時刻 | 3箇所以上 | □ |
| 五感描写 | 3箇所以上 | □ |
| 独自情報 | 1箇所以上 | □ |
| 体験談・エピソード | 1箇所以上 | □ |
| 失敗談・注意点 | 1箇所以上 | □ |
| 曖昧表現ゼロ | 「〜と言われています」等なし | □ |

## 💡 よくある質問

### Q: 文字数が足りない場合は？

**A**: 以下のセクションを充実させる
- 「パワースポットとしての特徴」を700文字まで拡充
- 「見どころ」で各スポットを詳しく描写（各200-300文字）
- 「周辺情報」を追加（グルメ、宿泊、他のスポット）

### Q: 独自性をどう出す？

**A**: 以下の情報を加える
- 地元の人だけが知る情報（裏参道、穴場の時間帯など）
- 失敗談（「夏の昼は避けるべき」など）
- 他のパワースポットとの比較
- 季節・時間帯による具体的な違い

### Q: 臨場感が出ない場合は？

**A**: 五感描写を追加
- **視覚**: 色、光、景色の具体的な描写
- **聴覚**: 音（鳥の声、せせらぎ、風など）
- **嗅覚**: 香り（梅、線香、潮の香りなど）
- **触覚**: 感触（風、温度、空気感など）
- **体感**: 「背筋が伸びる」「心が軽くなる」など

### Q: 事実と異なる情報を書いてしまわないか不安

**A**: 以下のルールを守る
- 不確かな情報は書かない
- 伝承・言い伝えは「〜という伝承があります」と明記
- 営業時間・料金は「要確認」と付記
- 体験談は「私の場合」と主観であることを明示

## 🎯 目指すべき記事像

### ❌ 避けるべき記事（低品質）
```
- ガイドブック的な情報の羅列
- 「美しい」「神秘的」などの曖昧な表現のみ
- 他のサイトと同じような内容
- 実用性に欠ける（時刻・所要時間などがない）
- 文字数が少ない（1,500文字未満）
```

### ✅ 目指すべき記事（高品質）
```
- 実際に訪れたかのような臨場感
- 具体的な数字・時刻が豊富
- 他にはない独自の視点・情報
- 読者が実際に訪問する際に役立つ実用情報
- 読んでいて「行ってみたい」と思わせる魅力
- 文字数2,500-3,500文字
```

## 📈 改善のコツ

**Before（低品質）:**

## 🔧 トラブルシューティング

### 問題: 書くことがなくなった

**解決策:**
1. Googleで他の記事を3つ読んで、それにない情報を考える
2. 以下を追加:
   - 失敗談・注意点
   - 季節ごとの違い
   - 時間帯ごとの違い
   - 地元グルメ・宿泊情報

### 問題: 文章が硬い・読みにくい

**解決策:**
1. 長い段落を2-3文ずつに分割
2. 箇条書きを適度に使う
3. 体験談を「〜でした」という過去形で書く
4. 「実は」「特に」などの接続詞を使う

### 問題: SEO対策が分からない

**解決策:**
1. タイトルに必須キーワードを含める:
   - スポット名
   - 「パワースポット」
   - 「ご利益」or「アクセス」
2. 見出しに自然にキーワードを散りばめる
3. 詰め込みすぎない（自然な文章を優先）
